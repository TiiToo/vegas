   136  	/**
   137  	 * Indicates the beginning of a new drawing path. If an open path exists (that is, if the current drawing position does not equal the previous position that is specified in a MovieClip.moveTo() method) and a fill is associated with it, that path is closed with a line and then filled. 
   138  	 * @param color A hexadecimal color value; for example, red is 0xFF0000, blue is 0x0000FF. If this value is not provided or is undefined, a fill is not created.
   139  	 * @param alpha [optional] An integer from 0 to 100 that specifies the alpha value of the fill. If this value is not provided, 100 (solid) is used. If the value is less than 0, Flash uses 0. If the value is greater than 100, Flash uses 100.
   140  	 */
   141  	public function beginFill(color:Number, alpha:Number):Void 
   142  	{
   143  		fa = isNaN(alpha) ? default_fc : alpha ;
   144  		fc = isNaN(color) ? default_fa : color ;
   145  		F.apply(_target, [fc, fa]) ;
   146  	}

   148  	/**
   149  	 * Indicates the beginning of a new drawing path. 
   150  	 * If the first parameter is undefined, or if no parameters are passed, the path has no fill. 
   151  	 * If an open path exists (that is if the current drawing position does not equal the previous position specified in a MovieClip.moveTo() method), and it has a fill associated with it, that path is closed with a line and then filled.
   152  	 * <p>This method fails if any of the following conditions exist:
   153  	 * <ul>
   154  	 * <li>The number of items in the colors, alphas, and ratios parameters are not equal.</li>
   155  	 * <li>The fillType parameter is not "linear" or "radial".</li>
   156  	 * <li>Any of the fields in the object for the matrix parameter are missing or invalid.</li>
   157  	 * </ul>
   158  	 * </p>
   159  	 * @param type Valid values are the string "linear" and the string "radial". You can use the {@code pegas.draw.FillType} enumerate static class to defined the value of this parameter.
   160  	 * @param colors An array of RGB hexadecimal color values you can use in the gradient; for example; red is 0xFF0000, blue is 0x0000FF. You can specify up to 15 colors. For each color, ensure to specify a corresponding value in the alphas and ratios parameters.
   161  	 * @param alphas An array of alpha values for the corresponding colors in the colors array; valid values are 0 to 100. If the value is less than 0, Flash uses 0. If the value is greater than 100, Flash uses 100.
   162  	 * @param ratios An array of color distribution ratios; valid values are 0 to 255. This value defines the percentage of the width where the color is sampled at 100%. Specify a value for each value in the colors parameter.
   163  	 * @param matrix A transformation matrix that can be in any one of three forms :
   164  	 * <p>A matrix object (supported by Flash Player 8 and later only), as defined by the flash.geom.Matrix class. The flash.geom.Matrix class includes a createGradientBox() method, which lets you conveniently set up the matrix for use with the beginGradientFill() method of MovieClip class. Macromedia recommends that you use this form of matrix for Flash Player 8 and later.</p>
   165  	 * <p>You can use the properties a, b, c, d, e, f, g, h, and i, which can be used to describe a 3 x 3 matrix of the following form:
   166  	 * {@code
   167  	 * a b c
   168  	 * d e f
   169  	 * g h i
   170  	 * }
   171  	 * </p>
   172  	 * <p>An object with the following properties: matrixType, x, y, w, h, r.</p>
   173  	 * <p>The properties indicate the following: matrixType is the string "box", x is the horizontal position relative to the registration point of the parent clip for the upper-left corner of the gradient, y is the vertical position relative to the registration point of the parent clip for the upper-left corner of the gradient, w is the width of the gradient, h is the height of the gradient, and r is the rotation in radians of the gradient.</p>
   174  	 * <p><b>Note :</b> For Flash Player 8 and later, Macromedia recommends that you define the matrix parameter in the form of a flash.geom.Matrix object (as described in the first item in this list).</p>
   175  	 */
   176  	public function beginGradientFill(type:String, colors:Array, alphas:Array, ratios:Array, matrix):Void 
   177  	{
   178  		GF.apply(_target, arguments) ;
   179  	}

   248  	/**
   249  	 * Specifies a line style that Flash uses for subsequent calls to the lineTo() and curveTo() methods until you call the lineStyle() method with different parameters. 
   250  	 * You can call lineStyle() in the middle of drawing a path to specify different styles for different line segments within a path.
   251  	 * <p><b>Note:</b> Calls to the clear() method set the value of line style back to undefined.</p>
   252  	 * @param thickness An integer that indicates the thickness of the line in points; valid values are 0 to 255. If a number is not specified, or if the parameter is undefined, a line is not drawn. If a value of less than 0 is passed, Flash Player uses 0. The value 0 indicates hairline thickness; the maximum thickness is 255. If a value greater than 255 is passed, the Flash interpreter uses 255.
   253  	 * @param color A hexadecimal color value of the line; for example, red is 0xFF0000, blue is 0x0000FF, and so on. If a value isn't indicated, Flash uses 0x000000 (black).
   254  	 * @param alpha An integer that indicates the alpha value of the line's color; valid values are 0 to 100. If a value isn't indicated, Flash uses 100 (solid). If the value is less than 0, Flash uses 0; if the value is greater than 100, Flash uses 100.
   255  	 * @param pixelHinting Added in Flash Player 8. A Boolean value that specifies whether to hint strokes to full pixels. This value affects both the position of anchors of a curve and the line stroke size itself. If a value is not indicated, Flash Player does not use pixel hinting.
   256  	 * @param noScale Added in Flash Player 8. A string that specifies how to scale a stroke. Valid values are as follows:
   257  	 * <p>
   258  	 * <ul>
   259  	 * <li>"normal" - Always scale the thickness (the default).</li>
   260  	 * <li>"none" - Never scale the thickness.</li>
   261  	 * <li>"vertical" - Do not scale thickness if object is scaled vertically only.</li>
   262  	 * <li>"horizontal" - Do not scale thickness if object is scaled horizontally only.</li>
   263  	 * </ul>
   264  	 * </p>
   265  	 * @param capsStyle Added in Flash Player 8. A string that specifies the type of caps at the end of lines. Valid values are: "round", "square", and "none". If a value is not indicated, Flash uses round caps.
   266  	 * @param jointStyle Added in Flash Player 8. A string that specifies the type of joint appearance used at angles. Valid values are: "round", "miter", and "bevel". If a value is not indicated, Flash uses round joints.
   267  	 * @param miterLimit Added in Flash Player 8. A number that indicates the limit at which a miter is cut off. Valid values range from 1 to 255 (and values outside of that range are rounded to 1 or 255). This value is only used if the jointStyle is set to "miter". If a value is not indicated, Flash uses 3. The miterLimit value represents the length that a miter can extend beyond the point at which the lines meet to form a joint. The value expresses a factor of the line thickness. For example, with a miterLimit factor of 2.5 and a thickness of 10 pixels, the miter is cut off at 25 pixels.
   268  	 */
   269  	public function lineStyle(thickness:Number, color:Number, alpha:Number, pixelHinting:Boolean, noScale:String, capsStyle:String, jointStyle:String, miterLimit:Number):Void 
   270  	{
   271  		t = isNaN(thickness) ? default_t : thickness ;
   272  		lc = isNaN(color) ? default_lc : color ;
   273  		la = isNaN(alpha) ? default_la : alpha ;
   274  		S.apply(_target, [t, lc, la].concat(arguments.slice(3)) ) ;
   275  	}
